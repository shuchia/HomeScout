# Snugd Beta Launch — "Polish & Ship" Design

**Goal:** Get Snugd into the hands of 10-20 friends & family within 1 month, validating the full funnel (search → favorite → compare → tour → decide) and willingness to pay for Pro features.

**Approach:** No new major features. Fix, polish, and add the minimum needed for a quality beta experience.

**Timeline:** ~3 weeks development, ~1 week manual testing and polish.

---

## 1. UI/UX Overhaul

### Brand Identity

- **Color palette:** Replace default Tailwind blue with a minimalist, warm palette. Muted teal or forest green as primary, neutral grays for structure, single accent color for CTAs. Clean and warm — Airbnb-meets-Notion, not corporate.
- **Typography:** Replace system font with a modern pairing — Inter or DM Sans for body, a slightly bolder display font for headings. Consistent sizing scale across all pages.
- **Logo:** Simple wordmark "snugd" in the display font with an optional subtle house/nest icon. Text-forward, no complex illustration.

### Mobile Responsiveness

- Bottom navigation bar (`BottomNav.tsx`) already exists — verify it works on all screen sizes.
- Search form: stack fields vertically on mobile, full-width inputs.
- Results grid: single column on mobile, 2 columns on tablet, 2 columns on desktop.
- Apartment cards: full-width on mobile with larger touch targets for favorite/compare buttons.
- Comparison bar: adapt for mobile (slide-up drawer or bottom sheet instead of floating bar).
- Tour detail page: tabbed interface should work on mobile already — verify and fix any overflow.

### Page-Level Polish

- Consistent spacing and padding across all pages.
- Better empty states (no results, no favorites, no tours) with helpful copy and subtle illustrations.
- Loading skeletons instead of spinners for content-heavy pages.
- Subtle animations: fade in search results, slide in cards on load.

## 2. Invite Code System

### User Flow

1. User signs up via Google OAuth → lands on app as free tier.
2. A subtle banner appears: "Have an invite code?" with an input field.
3. User enters code → backend validates → tier flips to Pro with a 90-day expiration.
4. Success animation + "You're on Pro!" message.
5. Invite code also accessible from Settings page and the Pricing page (alongside Stripe checkout).

### Database (Supabase migration)

New tables:

```sql
-- Invite codes generated by admin
create table public.invite_codes (
  code text primary key,
  max_uses integer not null default 1,
  times_used integer not null default 0,
  expires_at timestamptz,
  created_at timestamptz default now()
);

-- Track who redeemed what
create table public.invite_redemptions (
  id uuid default gen_random_uuid() primary key,
  code text references invite_codes(code) not null,
  user_id uuid references auth.users(id) on delete cascade not null,
  redeemed_at timestamptz default now(),
  unique(code, user_id)
);
```

Add column to profiles: `pro_expires_at timestamptz` — when set, Pro access expires at this timestamp.

### Backend Endpoints

- `POST /api/invite/redeem` — takes `{ "code": "BETA-ABC123" }`, validates (exists, not expired, uses remaining, not already redeemed by this user), updates `profiles.user_tier = 'pro'` and `profiles.pro_expires_at = now() + 90 days`, increments `times_used`.
- `GET /api/invite/status` — returns `{ "has_invite": true, "expires_at": "..." }` for the current user.
- `POST /api/admin/invite-codes` — generates codes with specified `max_uses` and optional `expires_at`.

### No changes to Stripe

Invite codes and Stripe are parallel paths to Pro. Existing payment flow stays as-is.

## 3. Guided Onboarding

### When It Triggers

- First sign-in only. Track `has_completed_onboarding` boolean in the `profiles` table.
- Lightweight tooltip/spotlight walkthrough — not a separate page or modal wizard.
- User can re-trigger from Settings ("Replay walkthrough").

### 4-Step Flow

1. **Search** — Spotlight on the search form: "Find apartments across 19 East Coast cities. Set your budget and preferences, and we'll match you."
2. **Favorites** — Spotlight on the heart icon area: "Save apartments you love. Compare them side by side."
3. **Tours** — Spotlight on the Tours nav item: "Plan your tours, capture notes and photos, get AI-powered insights."
4. **Pro Features** — Card with invite code input: "Unlock AI scoring, smart comparisons, and inquiry emails with a Pro invite code."

### Implementation

- Use `react-joyride` or a custom lightweight spotlight component (overlay + positioned tooltip).
- Each step: title, 1-2 sentence description, "Next" / "Skip" buttons.
- On completion or skip: set `has_completed_onboarding = true` in profiles.
- Step 4 includes the invite code input — if redeemed during onboarding, skip to a "Welcome to Pro!" confirmation.

### Promo Video Reuse

The 4-step walkthrough doubles as a script for a 30-60 second screen-recorded promo video. Record manually after the UI overhaul using the polished app. Not something we build — just a byproduct of the onboarding design.

## 4. Auth Bug Fix

### Problem

When a Supabase session token expires (1 hour), `getSession()` attempts a refresh. If the refresh fails silently, the `loading` state in `AuthContext` never resolves → blank white page. `localStorage.clear()` is the current workaround.

### Fix (in `AuthContext.tsx`)

- Add a 5-second timeout to auth initialization. If `getSession()` doesn't resolve, force `loading = false` and clear the stale session.
- Handle token refresh failure in `onAuthStateChange` — if the refresh event errors, sign the user out gracefully with a message.
- Show a visible error state: "Session expired. Please sign in again." instead of blank page.

### Scope

Small change, single file (`AuthContext.tsx`). High impact — prevents the #1 bug that makes testers think the app is broken.

## 5. Image Handling

### Problem

Apartment images come from scraped URLs (apartments.com CDN, etc.) which can break — hotlink blocking, expired URLs, removed images.

### Fix

- **Fallback image:** When an image fails to load, show a clean placeholder (gradient background with house icon and "No photo available" text) instead of a broken image icon.
- **Error handling:** Add `onError` handler to `<img>` tags in `ApartmentCard.tsx` and `ImageCarousel.tsx` that swaps to the fallback.
- **Next.js config:** Verify `next.config.ts` `images.remotePatterns` covers all domains scraped images come from, or use `unoptimized: true` for external URLs.

### Not in scope

S3 image caching/proxying — that's a scaling concern for after beta.

## 6. Feedback Widget

### What It Does

A small floating "Feedback" button in the bottom-right corner of every page.

### The Form

- **Type:** Quick toggle pills — Bug / Suggestion / General
- **Message:** Text area (required)
- **Screenshot:** Optional checkbox — uses `html2canvas` to auto-capture the current page
- **Context:** Automatically attaches page URL, user ID, tier, browser user agent (no user input needed)

### Storage

New Supabase table:

```sql
create table public.beta_feedback (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references auth.users(id) on delete cascade not null,
  type text not null check (type in ('bug', 'suggestion', 'general')),
  message text not null,
  screenshot_url text,
  page_url text,
  metadata jsonb default '{}',
  created_at timestamptz default now()
);
```

Review feedback directly in Supabase dashboard — no admin UI needed for 10-20 testers.

### Implementation

- Single `FeedbackWidget` component added to root layout.
- Hidden during onboarding walkthrough.
- Shows a "Thanks!" toast after submission.
- Screenshots uploaded to S3 (reuse existing photo upload infrastructure).

## 7. Bug Bash & Testing

### Happy Path (test on both mobile + desktop)

1. Sign up → onboarding walkthrough → redeem invite code
2. Search apartments → view results → favorite some
3. Add 2-3 to comparison → run AI comparison (Pro)
4. Start a tour from favorites → schedule → capture notes/photos
5. Voice capture → verify transcription
6. Generate inquiry email → day planner → decision brief
7. Submit feedback via widget
8. Sign out → sign back in (no blank page)

### Edge Cases

- Free user hitting 3/day search limit → proper messaging
- Free user trying Pro features → upgrade prompts shown
- Expired or invalid invite code → error message
- No search results for a city/budget combo → empty state
- Broken apartment images → fallback shown
- Mobile bottom nav, search form scrolling, tour detail tabs
- Slow network: loading skeletons, no duplicate form submissions

### Process

Manual testing over 2-3 days after all development is done. Fix anything that breaks. Not automated — this is the final polish pass before sharing with testers.
